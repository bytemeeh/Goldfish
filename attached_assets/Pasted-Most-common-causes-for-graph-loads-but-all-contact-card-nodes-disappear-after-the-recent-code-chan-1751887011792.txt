Most common causes for “graph loads but all contact-card nodes disappear” after the recent code changes

Likely culprit	Why it breaks nodes	One-minute check	Fix
1 nodesReady flag never flips	setNodesReady(true) runs only after setNodes(newNodes) inside the contacts → nodes useEffect. If contacts query fails or newNodes is [], the component stays in the “Loading contact graph…” branch and no nodes mount.	• In the browser console run window.__rf?.getNodes?.().length — it will be 0.• Network tab shows /api/contacts 4xx/5xx.	Make sure /api/contacts returns 200 with array.  Log the fetch error and show toast instead of blocking graph.
2 ID casting produced duplicates / nulls	We now cast id: String(contact.id). If the DB still contains numeric IDs for older rows and some new UUID rows, duplicates arise ('7' vs '7'). React-Flow drops every duplicate key silently.	console.table(nodes.map(n=>n.id)) after mapping; look for repeats or “null”.	Migrate legacy rows to UUID; enforce NOT NULL and uniqueness; or prefix integers ("legacy-"+id).
3 nodeTypes became undefined	Moving const nodeTypes = { contact: ContactNode } above the ContactNode import leaves value undefined at first evaluation → React-Flow renders nothing, no warning.	console.log(nodeTypes) just before <ReactFlow> — should output an object, not “undefined”.	Declare nodeTypes after the ContactNode import (or in same file but below import lines).
4 ContactNode wrapper now absolute/hidden	In ContactNode.tsx we added Tailwind classes; one typo like absolute without coordinates or hidden causes zero-size/hidden node.	Inspect .react-flow__node in DevTools; if width/height = 0 or display:none, it’s CSS.	Remove the errant class; keep wrapper relative with fixed width/height.
5 Positions became NaN	Collision-resolver returns position: { x: undefined, y: undefined }. React-Flow ignores nodes whose translate() contains NaN.	console.log(nodes[0].position) — values must be numbers.	Provide fallback (0,0) or trace missing calculations in buildHierarchy.
6 Graph provider missing	In refactor you wrapped graph with extra div and accidentally dropped <ReactFlowProvider>. Without provider React-Flow mounts but renders nothing.	Components tree in React-DevTools lacks ReactFlowProvider.	Re-add <ReactFlowProvider> around <ReactFlow>.
7 StrictMode double-render wipes state	In dev build only, <React.StrictMode> remounts and resets nodes right after you set them; nothing left to render.	Issue occurs in dev but not in production bundle.	Temporarily remove React.StrictMode; ensure state initialisation resilient to double mount.
8 API now returns parent_id integers	Mapping expects parentId UUID string; undefined parent breaks parentNode link and layout algorithm aborts.	Inspect one fetched contact JSON.	Adjust mapping: parentNode: contact.parentId ? String(contact.parentId) : undefined.
9 CSS overlay hides canvas	A new full-screen div (debug logger / snack bar) with high z-index covers the canvas; nodes exist but are hidden.	Toggle overlay visibility in Elements; nodes appear underneath.	Lower z-index or set pointer-events:none on overlay.
10 React-Flow fit-view call zooms to infinity	If you call fitView({ padding: 0.1 }) before nodes exist, it may zoom to 0 and later not re-fit.	Check viewport inspector; zoom shows 0.	Invoke fitView in onLoad after verifying nodes.length > 0.
