Likeliest causes for “nodes no longer visible” after the recent code-round

Suspect	Why the change round can trigger it	Quick verification	Remedy
1. nodeTypes becomes undefined	Moving const nodeTypes = { contact: ContactNode } above the ContactNode import (or before the component is defined) leaves the value undefined; React-Flow renders nothing without complaining.	console.log(nodeTypes) right before <ReactFlow …> – should print { contact: ƒ }.	Ensure the ContactNode symbol is in scope before the object literal, or move nodeTypes immediately after the import line.
2. ID duplicates / falsy IDs	We cast id: String(contact.id), but if some rows now have null IDs (migration mismatch) or duplicates, React-Flow silently drops nodes with non-unique keys.	console.table(nodes.map(n=>n.id)) – check for null, undefined, duplicate UUIDs.	Fix seed data; ensure contact.id is non-null UUID, add whereNotNull in query, or regenerate demo rows.
3. Positions became NaN / undefined	When converting IDs to strings we may have accidentally written position: { x: undefined, y: undefined }. React-Flow skips rendering nodes with invalid transforms.	Inspect one node obj in devtools – look at position.	Provide sane default ({x:0,y:0}) or carry over original coords.
4. CSS over-zealous (display:none, zero size, opacity 0)	Added highlight classes; a typo like hidden or opacity-0 in wrapper hides content.	Temporarily add outline:1px solid red to .react-flow__node in DevTools; if outline shows but card invisible, inner div has zero height/width or opacity.	Remove/adjust offending Tailwind class.
5. Wrapper now position:absolute	In contact node we may have added absolute without top/left, so it sits at (0,0) with zero area until transform updates.	Inspect computed style: if position:absolute and width:0, that’s the issue.	Replace with relative or set width/height.
6. React-Flow not mounted	Accidentally removed <ReactFlowProvider> or returned early (e.g., if (!nodes.length) return null).	React DevTools: absence of <div class="react-flow"> element.	Re-wrap in provider; ensure component returns graph even when empty array.
7. Query failed, nodes array empty	New backend UUIDs but frontend still calls /api/contacts and receives 404/500; state hook initialises to [], so nothing to paint.	Network tab: does GET /api/contacts return 200 and JSON list?	Fix backend error; adjust fetch; log error.
8. SVG layer covers nodes	Drop-target ring (ring-4) may render on parent div with absolute w-full h-full and high z-index, hiding nodes.	Toggle layer visibility in Elements panel; if nodes appear, z-index issue.	Lower z-index or set pointer-events:none on overlay.
9. Strict Mode double-mount killed local state	If you added React.StrictMode back, setNodes state may reset after first render, yielding zero nodes.	DevTools Components panel shows two mounts/unmounts.	Remove StrictMode wrapper in dev build.
10. Import path break	Moving files renamed module; build succeeds due to TS path alias but hot reload drops component, yielding blank nodes.	Console shows “Failed to compile” or “Cannot find module …” warnings.	Fix import path / case sensitivity.
